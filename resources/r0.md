# `learn-claude-code` 项目深度解析

`learn-claude-code` 是一个**从0到1构建类似 Claude Code 的 nano 级 AI Agent** 的教学型开源项目。它通过 12 个渐进式的 Session（阶段），用极简的 Python 代码，层层拆解 AI Agent 的核心机制，让学习者真正理解 Agent 内部是如何工作的。

---

## 一、项目核心定位与设计理念
### 1. 核心目标
- **不是生产级工具**：而是一个**学习项目**，故意简化/省略了生产级 Agent 的复杂机制（如完整事件总线、权限治理等）。
- **聚焦原理**：通过“每次只加一个机制”的方式，让学习者从最基础的循环，逐步理解到多智能体协同、任务隔离等高级功能。

### 2. 核心设计思想
项目的核心可以用一句话概括：**“The model is the agent. Our job is to give it tools and stay out of the way.”**（模型即 Agent。我们的工作是给它工具，然后别挡道。）

---

## 二、项目核心架构：The Agent Pattern
这是所有 AI Agent 的**最小循环**，也是整个项目的基石。文档中用 ASCII 图清晰展示了这个逻辑：

```
User --> messages[] --> LLM --> response
                                  |
                        stop_reason == "tool_use"?
                       /                          \
                     yes                           no
                      |                             |
                execute tools                    return text
                append results
                loop back -----------------> messages[]
```

### 循环拆解（核心中的核心）
1. **输入**：用户的请求被放入 `messages[]` 数组（对话历史）。
2. **LLM 调用**：将 `messages[]` 传给 LLM，获取响应。
3. **判断停止原因**：
   - 如果 `stop_reason != "tool_use"`：LLM 直接返回文本，循环结束。
   - 如果 `stop_reason == "tool_use"`：LLM 要求调用工具，进入下一步。
4. **工具执行**：遍历 LLM 响应中的工具调用请求，执行对应的工具函数，获取结果。
5. **结果追加**：将工具执行结果作为“用户消息”追加到 `messages[]` 中。
6. **循环回去**：再次调用 LLM，让它根据工具结果继续处理。

### 核心代码实现（极简版）
项目中给出了这个循环的 Python 代码，所有后续 Session 都是在这个基础上**叠加机制，不改变循环本身**：

```python
def agent_loop(messages):
    while True:
        # 1. 调用 LLM
        response = client.messages.create(
            model=MODEL, system=SYSTEM,
            messages=messages, tools=TOOLS,
        )
        # 2. 追加 LLM 响应到对话历史
        messages.append({"role": "assistant",
                         "content": response.content})

        # 3. 判断是否需要调用工具
        if response.stop_reason != "tool_use":
            return  # 不需要，直接返回

        # 4. 执行工具
        results = []
        for block in response.content:
            if block.type == "tool_use":
                # 调用对应的工具处理函数
                output = TOOL_HANDLERS[block.name](**block.input)
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": output,
                })
        # 5. 追加工具结果到对话历史，循环回去
        messages.append({"role": "user", "content": results})
```

## 三、项目目录结构
```
learn-claude-code/
|
|-- agents/                        # Python 参考实现（s01-s12 + s_full 完整版）
|-- docs/{en,zh,ja}/               # 心智模型优先的文档（3种语言）
|-- web/                           # 交互式学习平台（Next.js）
|-- skills/                        # s05 用到的技能文件
+-- .github/workflows/ci.yml      # CI：类型检查 + 构建
```

### 关键目录说明
- **`agents/`**：核心代码区，包含 12 个 Session 的逐步实现，以及一个整合所有机制的 `s_full.py`。
- **`docs/`**：文档区，采用“心智模型优先”的写法：先讲问题，再给解决方案，配 ASCII 图，最后给极简代码。支持中、英、日三种语言。
- **`web/`**：交互式学习平台，包含可视化步骤图、源码查看器和文档，本地运行后可以直观看到 Agent 的工作流程。

## 四、12 个渐进式 Session 详解
项目的核心是 12 个 Session，每个 Session 只加一个机制，配一个“座右铭”（Motto）帮助理解核心思想。

### Phase 1: The Loop（基础循环）
| Session | 主题 | 座右铭 | 核心内容 |
| :--- | :--- | :--- | :--- |
| **s01** | The Agent Loop | *One loop & Bash is all you need* | 实现最基础的 Agent 循环 + 一个 Bash 工具，证明“一个循环+一个工具就是 Agent”。 |
| **s02** | Tool Use | *Adding a tool means adding one handler* | 保持循环不变，引入“调度映射表”（`dispatch map`）：工具名 -> 处理函数，方便添加新工具。 |

### Phase 2: Planning & Knowledge（规划与知识）
| Session | 主题 | 座右铭 | 核心内容 |
| :--- | :--- | :--- | :--- |
| **s03** | TodoWrite | *An agent without a plan drifts* | 引入“先列计划，再执行”的机制：让 LLM 先写 Todo 列表，再按步骤执行，任务完成率翻倍。 |
| **s04** | Subagents | *Break big tasks down; each subtask gets a clean context* | 引入“子 Agent”：大任务拆成小任务，每个子任务用独立的 `messages[]`，保持主对话干净。 |
| **s05** | Skills | *Load knowledge when you need it, not upfront* | 引入“技能”：通过 `tool_result` 注入知识（如 `SKILL.md`），而不是全塞在 System Prompt 里。 |
| **s06** | Context Compact | *Context will fill up; you need a way to make room* | 引入“三层上下文压缩策略”：解决上下文过长的问题，实现无限会话。 |

### Phase 3: Persistence（持久化）
| Session | 主题 | 座右铭 | 核心内容 |
| :--- | :--- | :--- | :--- |
| **s07** | Tasks | *Break big goals into small tasks, order them, persist to disk* | 引入“基于文件的任务图”：带依赖关系的任务，持久化到磁盘，为多 Agent 协作打基础。 |
| **s08** | Background Tasks | *Run slow operations in the background; the agent keeps thinking* | 引入“后台任务”：用守护线程运行慢操作，完成后注入通知，Agent 不用傻等。 |

### Phase 4: Teams（团队协作）
| Session | 主题 | 座右铭 | 核心内容 |
| :--- | :--- | :--- | :--- |
| **s09** | Agent Teams | *When the task is too big for one, delegate to teammates* | 引入“持久化队友 + 异步邮箱”：用 JSONL 格式的邮箱实现 Agent 间通信。 |
| **s10** | Team Protocols | *Teammates need shared communication rules* | 引入“请求-响应模式”：用一个统一的协议驱动所有协商（如关机、计划审批）。 |
| **s11** | Autonomous Agents | *Teammates scan the board and claim tasks themselves* | 引入“自主认领”：Agent 空闲时自动扫描任务板，自己认领任务，不需要领导分配。 |
| **s12** | Worktree + Task Isolation | *Each works in its own directory, no interference* | 引入“工作树隔离”：每个任务在自己的目录里工作，互不干扰；任务管理目标，工作树管理目录。 |

## 五、项目的“Scope”（边界）
文档特别强调：这是一个**0->1 的学习项目**，为了教学目的，**故意简化或省略了以下生产级机制**：
1. 完整的事件/钩子总线（如 `PreToolUse`、`SessionStart/End`、`ConfigChange`）。
2. 基于规则的权限治理和信任工作流。
3. 会话生命周期控制（恢复/分叉）和高级工作树生命周期控制。
4. 完整的 MCP 运行时细节（传输/OAuth/资源订阅/轮询）。
5. 团队 JSONL 邮箱协议仅作为教学实现，不是生产级内部机制。

## 六、快速开始（动手跑起来）
1. **克隆项目**：
   ```bash
   git clone https://github.com/shareAI-lab/learn-claude-code
   cd learn-claude-code
   ```
2. **安装依赖**：
   ```bash
   pip install -r requirements.txt
   ```
3. **配置 API Key**：
   ```bash
   cp .env.example .env
   # 编辑 .env，填入你的 ANTHROPIC_API_KEY
   ```
4. **运行 Session**：
   ```bash
   # 从 s01 开始（最基础的循环）
   python agents/s01_agent_loop.py
   
   # 运行完整版本（所有机制整合）
   python agents/s_full.py
   ```
5. **启动交互式 Web 平台**：
   ```bash
   cd web && npm install && npm run dev
   # 访问 http://localhost:3000
   ```

## 七、学完之后能做什么？
项目给出了两个延伸方向：
1. **Kode Agent CLI**：一个开源的编码 Agent CLI，支持 Windows，可插拔 GLM/MiniMax/DeepSeek 等开源模型。
2. **Kode Agent SDK**：一个独立的库，没有“每个用户一个终端进程”的开销，可以嵌入后端、浏览器扩展、嵌入式设备等。

此外，还有一个姐妹 Repo `claw0`，它在同样的 Agent 核心上，加入了“心跳（Heartbeat）”和“定时任务（Cron）”，把 Agent 从“用完即丢”变成“每30秒醒来看有没有活干”的常驻助手。

---

## 总结
`learn-claude-code` 的价值在于**“心智模型优先”**：它不追求做一个功能强大的工具，而是通过极简的代码和渐进的步骤，让你真正理解“Agent 到底是怎么工作的”。一旦理解了这 12 个 Session，你就能把这些机制迁移到任何场景（比如你想做的《会计数据与智能分析》Agent）。
