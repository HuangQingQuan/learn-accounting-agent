# S1: 项目初始化与环境搭建

## 目标
完成 learn-accounting-agent 项目的目录创建、Python环境配置、依赖安装，为后续开发奠定基础。

## 前置条件
- Python 3.9+ 已安装
- Git 已安装
- 代码编辑器（推荐 VS Code）

## 操作步骤

### 1. 项目目录创建
```bash
# 创建项目根目录
mkdir learn-accounting-agent
cd learn-accounting-agent

# 创建子目录结构
mkdir docs agents skills examples
mkdir agents/utils skills/impl
```

### 2. 虚拟环境初始化
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境（Windows）
venv\Scripts\activate

# 激活虚拟环境（Linux/Mac）
source venv/bin/activate
```

### 3. 依赖安装
```bash
# 安装项目依赖
pip install -r requirements.txt

# 验证安装
pip list
```

### 4. 项目结构验证
项目应具有以下目录结构：
```
learn-accounting-agent/
├── docs/                # 分步骤教学文档
├── agents/              # 智能体核心代码
│   └── utils/           # 工具函数
├── skills/              # 技能说明+实现
│   └── impl/            # 技能代码实现
├── examples/            # 示例数据+演示代码
├── requirements.txt     # 依赖包列表
└── README.md           # 项目说明
```

## 环境要求说明

### Python 版本要求
- **最低版本**: Python 3.9
- **推荐版本**: Python 3.11+
- **原因**: 需要支持现代类型注解和异步编程特性

### 核心依赖说明
- **pandas**: 数据处理和分析，用于账务数据解析
- **openpyxl**: Excel文件读写支持
- **sqlalchemy**: 数据库ORM，支持多种数据库
- **fastapi**: Web框架，用于API接口开发
- **langchain**: LLM集成框架
- **jinja2**: 模板引擎，用于报告生成

## 常见问题

### Q1: 虚拟环境激活失败
**解决方案**: 
- Windows: 确保使用 `venv\Scripts\activate`
- Linux/Mac: 确保使用 `source venv/bin/activate`
- 检查Python路径是否正确

### Q2: 依赖安装失败
**解决方案**:
- 升级pip: `pip install --upgrade pip`
- 使用国内镜像: `pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt`
- 检查Python版本兼容性

### Q3: 权限问题
**解决方案**:
- Windows: 以管理员身份运行命令提示符
- Linux/Mac: 使用 `sudo` 或检查文件权限
