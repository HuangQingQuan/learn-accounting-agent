## 核心架构设计

### 1. 智能体基类设计思路

#### 1.1 设计原则
- **单一职责**: 基类只负责通用能力，具体业务由子类实现
- **开放封闭**: 对扩展开放，对修改封闭
- **依赖倒置**: 依赖抽象而非具体实现
- **接口隔离**: 技能接口设计简洁明确

#### 1.2 核心能力
- **技能注册管理**: 动态注册、注销、查询技能
- **任务调度机制**: 根据任务类型匹配对应技能
- **状态管理**: 跟踪智能体使用状态和统计信息
- **异常处理**: 统一的异常处理和错误记录

### 2. 代码实现详解

#### 2.1 BaseAgent 基类实现

**文件位置**: `agents/base_agent.py`

```python
from abc import ABC, abstractmethod
from typing import Dict, Any, List, Optional, Callable
from datetime import datetime
import logging

class BaseAgent(ABC):
    """智能体基类：定义通用能力"""
    
    def __init__(self, name: str = "BaseAgent", skills: Dict[str, Any] = None):
        self.name = name
        self.skills = skills or {}
        self.logger = logging.getLogger(f"{__name__}.{name}")
        self.created_at = datetime.now()
        self.last_used = None
        self.usage_count = 0
```

**关键特性说明**:
- **抽象基类**: 使用ABC确保子类必须实现核心方法
- **技能注册表**: 使用字典存储技能函数，支持动态管理
- **日志记录**: 为每个智能体实例创建独立日志器
- **使用统计**: 跟踪创建时间、最后使用时间、使用次数

#### 2.2 技能管理机制

```python
def register_skill(self, skill_name: str, skill_func: Callable) -> None:
    """注册技能"""
    if skill_name in self.skills:
        raise ValueError(f"技能 '{skill_name}' 已存在")
    self.skills[skill_name] = skill_func
    self.logger.info(f"技能 '{skill_name}' 注册成功")

def unregister_skill(self, skill_name: str) -> None:
    """注销技能"""
    if skill_name not in self.skills:
        raise KeyError(f"技能 '{skill_name}' 不存在")
    del self.skills[skill_name]
    self.logger.info(f"技能 '{skill_name}' 注销成功")
```

**设计优势**:
- **动态管理**: 运行时可以灵活添加或移除技能
- **冲突检测**: 防止技能名称重复
- **操作记录**: 所有技能操作都有日志记录

#### 2.3 抽象方法定义

```python
@abstractmethod
def run(self, task: str, data: Any, **kwargs) -> Any:
    """执行核心任务（子类必须实现）"""
    pass
```

**强制实现**: 确保所有子类都必须定义自己的任务执行逻辑。

#### 3.2 任务执行逻辑

```python
def run(self, task: str, data: Any, **kwargs) -> Any:
    """执行账务审核任务"""
    # 更新使用统计
    self._update_usage()
    
    # 验证任务类型
    if not self.validate_task(task):
        raise TaskError(f"不支持的任务类型：{task}")
    
    # 执行技能并记录
    try:
        result = self.skills[task](data, **kwargs)
        self._record_success(task, result)
        return result
    except Exception as e:
        self._record_failure(task, e)
        raise SkillError(f"技能执行失败：{str(e)}") from e
```

**执行流程**:
1. **统计更新**: 更新使用次数和时间
2. **任务验证**: 检查任务类型是否支持
3. **技能执行**: 调用对应的技能函数
4. **结果记录**: 记录执行结果和统计信息
5. **异常处理**: 统一的异常处理和记录

### 4. 配置管理系统

#### 4.1 AgentConfig 配置类

**文件位置**: `agents/config.py`

```python
@dataclass
class AgentConfig:
    """智能体配置类"""
    
    # 基础配置
    name: str = "AccountingAgent"
    version: str = "1.0.0"
    debug: bool = False
    
    # LLM配置
    llm_provider: str = "openai"
    openai_api_key: Optional[str] = None
    openai_model: str = "gpt-3.5-turbo"
    
    # 审核配置
    max_batch_size: int = 1000
    default_rules: Dict[str, Any] = field(default_factory=lambda: {
        "max_amount": 100000,
        "forbidden_subjects": ["违规科目1", "违规科目2"],
        "duplicate_check": True
    })
```

**配置特性**:
- **类型安全**: 使用dataclass确保类型正确性
- **默认值**: 提供合理的默认配置
- **环境变量**: 支持从环境变量加载配置
- **文件存储**: 支持配置的保存和加载

## 常见问题

### Q1: 如何处理技能执行异常？
**解决方案**: 
- 使用try-catch包装技能执行
- 记录详细的错误信息和堆栈跟踪
- 提供统一的异常处理机制

### Q2: 技能函数的签名有什么要求？
**解决方案**: 
- 技能函数必须接受data参数
- 可以接受额外的**kwargs参数
- 返回值可以是任意类型

### Q3: 如何扩展智能体功能？
**解决方案**: 
- 继承BaseAgent或AccountingAgent
- 重写run方法实现自定义逻辑
- 添加新的属性和方法

### Q4: 配置文件如何管理？
**解决方案**: 
- 使用JSON格式存储配置
- 支持环境变量覆盖
- 提供配置验证和默认值
